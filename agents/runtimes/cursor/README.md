# Runtime: Cursor (secondary)

Cursor is optional. Product law remains each repo's `AGENTS.md`.

| Path | Role |
|------|------|
| `~/.cursor/` | Tool home |
| `~/.cursor/hooks.json` | Shared PreToolUse-style hook wiring (symlink into pack) |
| `~/.cursor/hooks/guard.sh` | Portable command guard (symlink into pack) |
| `~/.cursor/skills` | Shared portable skills (symlink into pack) |
| Repo `.cursor/rules` | Optional IDE rules - must not fork product law |

## Autonomy

Cursor UI permission settings are vendor-managed. The portable pack still installs global hooks so the shared command guard runs when Cursor supports hook execution.

Prefer fixing `AGENTS.md` over growing Cursor-only rule forks.

Cursor reads the open `AGENTS.md` format at a project root and supports `SKILL.md` Agent Skills. The installer gives Cursor the same shared skill source as Codex, Claude, and Grok. Cursor user-interface rules remain vendor-managed and must not become a second copy of machine law.
